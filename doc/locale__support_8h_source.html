46  * How many times a string identifier can grow (in bytes size) when
47  * performing lower / upper on DB identifiers.
48  * This growing can occur only in UTF-8 charset (see Unicode data, for example
49  * lower case for U+023A is U+2C65 - 2 bytes to 3 bytes).
50  * This restriction does not apply to user strings.
51  * The Turkish rules for casing do not apply on identifiers, only for user
52  * strings. Identifier casing rules are the same for locales, assuming they
53  * use the same charset and Unicode data (built-in en_US.utf8,ko_KR.utf8 and
54  * tr_TR.utf8 have different Unicode data then LDML de_DE.utf8, hence they
55  * produce different identifier casing rules).
56  */

59 /* Mask for next sequence. Used to determine next string in sorting order
60  * in LIKE operator.
61  * If the value of 'next' has this bit set, then the next sequence is a
62  * contraction, and the lower part of value indicates the contraction id
63  * Otherwise, the 'next' value indicates a Unicode codepoint */
