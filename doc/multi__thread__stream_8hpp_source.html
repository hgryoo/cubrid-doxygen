41  /* stream with capability to write/read concurrently
42  * the read, read_partial, read_serial, write require a function argument to perform custom write/read operation;
43  * Interface:
44  *
45  * write (amount, write_function):
46  * - the amount needs to be pre-computed
47  * - internally, the write_function is wrapped by a reserve/commit pair
48  *
49  * read (pos, amount, read_function):
50  * - the position and amount of data are required
51  * - internally, the read_function is wrapped by start_read, end_read (for latching the page in buffer)
52  * - if required range is still in bip_buffer storage and it spans across the its boundary, the read
53  * creates internally a buffer and provides a copy of range to read_function
54  *
55  * read_serial (amount, read_function):
56  * - the stream stores an internal position for serial reading
57  * - this method must be used by only one thread
58  *
59  * read_partial (pos, amount, read_amount, read_function):
60  * - similar with read, but it provides only 'read_amount' in case of range spans across buffer boundary
61  *
62  * Internals:
63  * The reserved position are stored in a circular queue of stream_reserve_context elements.
64  * The memory storage is implemented using bip_buffer.
65  * reserve_with_buffer/commit_append : internal methods used by write;
66  * get_data_from_pos/unlatch_read_data : internal methods used by read;
67  * wait_for_data : used by read when requested range is not yet produced
68  */
