155  // cubthread::entry
156  //
157  // description
158  // this is the thread context used by most server module functions to access thread-specific information quickly
159  //
160  // note
161  // in CUBRID, thread entries are pooled and dispatched by cubthread::manager. see thread_manager.hpp for details.
162  //
163  // this is an implementation in progress. for backward compatibility, all legacy members in this class are public;
164  // however, they will be gradually converted into private members with access functions
165  //
166  // this header is paired with thread_compat.hpp which is used for compatibility between modules. only server
167  // module (with its SERVER_MODE and SA_MODE versions) has access to the content of this entry. client modules only
168  // sees a void pointer.
169  //
170  // to avoid major refactoring, the THREAD_ENTRY alias is kept
