445 /* This structure is filled in by the XML_UnknownEncodingHandler to
446  provide information to the parser about encodings that are unknown
447  to the parser.
448 
449  The map[b] member gives information about byte sequences whose
450  first byte is b.
451 
452  If map[b] is c where c is >= 0, then b by itself encodes the
453  Unicode scalar value c.
454 
455  If map[b] is -1, then the byte sequence is malformed.
456 
457  If map[b] is -n, where n >= 2, then b is the first byte of an
458  n-byte sequence that encodes a single Unicode scalar value.
459 
460  The data member will be passed as the first argument to the convert
461  function.
462 
463  The convert function is used to convert multibyte sequences; s will
464  point to a n-byte sequence where map[(unsigned char)*s] == -n. The
465  convert function must return the Unicode scalar value represented
466  by this byte sequence or -1 if the byte sequence is malformed.
467 
468  The convert function may be NULL if the encoding is a single-byte
469  encoding, that is if map[b] >= -1 for all bytes b.
470 
471  When the parser is finished with the encoding, then if release is
472  not NULL, it will call release passing it the data member; once
473  release has been called, the convert function will not be called
474  again.
475 
476  Expat places certain restrictions on the encodings that are supported
477  using this mechanism.
478 
479  1. Every ASCII character that can appear in a well-formed XML document,
480  other than the characters
481 
482  $@\^`{}~
483 
484  must be represented by a single byte, and that byte must be the
485  same byte that represents that character in ASCII.
486 
487  2. No character may require more than 4 bytes to encode.
488 
489  3. All characters encoded must have Unicode scalar values <=
490  0xFFFF, (i.e., characters that would be encoded by surrogates in
491  UTF-16 are not allowed). Note that this restriction doesn't
492  apply to the built-in support for UTF-8 and UTF-16.
493 
494  4. No Unicode character may be encoded by more than one distinct
495  sequence of bytes.
496 */
497  typedef struct
498  {
499  int map[256];
500  void *data;
501  int (XMLCALL * convert) (void *data, const char *s);
502  void (XMLCALL * release) (void *data);
503  } XML_Encoding;
