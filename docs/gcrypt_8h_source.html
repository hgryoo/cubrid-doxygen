466 /* Extract MPIs from an s-expression using a list of parameters. The
467  * names of these parameters are given by the string LIST. Some
468  * special characters may be given to control the conversion:
469  *
470  * + :: Switch to unsigned integer format (default).
471  * - :: Switch to standard signed format.
472  * / :: Switch to opaque format.
473  * & :: Switch to buffer descriptor mode - see below.
474  * ? :: The previous parameter is optional.
475  *
476  * In general parameter names are single letters. To use a string for
477  * a parameter name, enclose the name in single quotes.
478  *
479  * Unless in gcry_buffer_t mode for each parameter name a pointer to
480  * an MPI variable is expected that must be set to NULL prior to
481  * invoking this function, and finally a NULL is expected. Example:
482  *
483  * _gcry_sexp_extract_param (key, NULL, "n/x+ed",
484  * &mpi_n, &mpi_x, &mpi_e, NULL)
485  *
486  * This stores the parameter "N" from KEY as an unsigned MPI into
487  * MPI_N, the parameter "X" as an opaque MPI into MPI_X, and the
488  * parameter "E" again as an unsigned MPI into MPI_E.
489  *
490  * If in buffer descriptor mode a pointer to gcry_buffer_t descriptor
491  * is expected instead of a pointer to an MPI. The caller may use two
492  * different operation modes: If the DATA field of the provided buffer
493  * descriptor is NULL, the function allocates a new buffer and stores
494  * it at DATA; the other fields are set accordingly with OFF being 0.
495  * If DATA is not NULL, the function assumes that DATA, SIZE, and OFF
496  * describe a buffer where to but the data; on return the LEN field
497  * receives the number of bytes copied to that buffer; if the buffer
498  * is too small, the function immediately returns with an error code
499  * (and LEN set to 0).
500  *
501  * PATH is an optional string used to locate a token. The exclamation
502  * mark separated tokens are used to via gcry_sexp_find_token to find
503  * a start point inside SEXP.
504  *
505  * The function returns 0 on success. On error an error code is
506  * returned, all passed MPIs that might have been allocated up to this
507  * point are deallocated and set to NULL, and all passed buffers are
508  * either truncated if the caller supplied the buffer, or deallocated
509  * if the function allocated the buffer.
510  */
