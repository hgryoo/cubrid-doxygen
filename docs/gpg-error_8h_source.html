61 /* The GnuPG project consists of many components. Error codes are
62  exchanged between all components. The common error codes and their
63  user-presentable descriptions are kept into a shared library to
64  allow adding new error codes and components without recompiling any
65  of the other components. The interface will not change in a
66  backward incompatible way.
67 
68  An error code together with an error source build up an error
69  value. As the error value is been passed from one component to
70  another, it preserver the information about the source and nature
71  of the error.
72 
73  A component of the GnuPG project can define the following macros to
74  tune the behaviour of the library:
75

77  gpg_err_source_t to make that source the default for gpg_error().
78  Otherwise GPG_ERR_SOURCE_UNKNOWN is used as default.
79 
80  GPG_ERR_ENABLE_GETTEXT_MACROS: Define to provide macros to map the
81  internal gettext API to standard names. This has only an effect on
82  Windows platforms.
83 
84  GPGRT_ENABLE_ES_MACROS: Define to provide "es_" macros for the
85  estream functions.
