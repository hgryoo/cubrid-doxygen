# CUBRID Engine API Reference

Auto-generated source-level documentation for the
[CUBRID](https://github.com/CUBRID/cubrid) database engine
(`develop` branch).

This site is rebuilt **weekly** from the latest CUBRID source by a
GitHub Actions workflow.

## How to use this site

- **Files** &mdash; browse documentation by source file path under
  [API Reference → Files](cubrid/files.md).
- **Classes** &mdash; navigate the C++ class hierarchy from
  [Classes](cubrid/classes.md).
- **Namespaces** &mdash; jump into `cubrid::*` and other namespaces
  from [Namespaces](cubrid/namespaces.md).
- **Modules** &mdash; logical groupings (`@defgroup`) from
  [Modules](cubrid/modules.md).
- **Search** &mdash; use the search box at the top right (instant,
  client-side, fuzzy-matched).

## What is documented

The reference covers the CUBRID server engine source tree
(`cubrid/src/`) excluding the client interface drivers
(`cci`, `jdbc`, `odbc`, `oledb`) and the management/admin tools
(`cm_common`, `tools`, `win_tools`).

| Item | Setting |
| --- | --- |
| Upstream repository | [CUBRID/cubrid](https://github.com/CUBRID/cubrid) |
| Branch tracked | `develop` |
| Rebuild cadence | Weekly (Sundays UTC) + manual trigger |
| Generator | Doxygen → MkDocs Material via [mkdoxy](https://github.com/JakubAndrysek/mkdoxy) |

## Build diagnostics

Doxygen warnings from the most recent build are recorded at
[Build Info → Doxygen Warnings](build/warnings.md).

## Reporting documentation issues

If a symbol is missing, mis-rendered, or links to the wrong place,
open an issue on the
[cubrid-doxygen](https://github.com/CUBRID/cubrid-doxygen) repository.
For issues in the underlying source comments, open an issue on
[CUBRID/cubrid](https://github.com/CUBRID/cubrid) instead.
